### What Is Poorly Structured

#### `app.py` — Monolithic God Script

```
app.py (296 lines):
  ├── hardcoded path (line 8)        ← deployment killer
  ├── process_tif()                  ← mixed concerns
  ├── 180 lines of HTML string       ← presentation in logic layer
  ├── f-string CSS injection         ← major security risk
  └── webbrowser.open()              ← not a web server
```

**Specific Problems:**

1. **Line 8:** `DATASET_PATH = r"C:\Users\sandi\Minor Project\sundarbans\sundarbans"` — hardcoded absolute path on a Windows machine. This script breaks on every other machine and in every deployment environment.

2. **Lines 99–285:** Entire HTML is a Python f-string. Any value containing `{` or `}` will crash with a `KeyError`. The `chart_data` injection on line 278 (`data: {chart_data}`) embeds a raw Python list — this only works by accident because Python's list `repr` happens to be valid JSON.

3. **Lines 62–91:** HTML generation via string concatenation with no escaping. A filename like `GMW_N00<script>alert(1)</script>.tif` would execute JavaScript (XSS).

4. **Carbon model:** `carbon = area_ha * 100` (line 20) is a made-up constant. The v3 pipeline uses `* 1000`. The two files use **different constants with no documentation**. This is a scientific credibility issue for any demo.

5. **Risk classification uses absolute carbon stock** (lines 36–41 in `app.py`), not change detection. A region with 1,001 carbon tons is "Moderate" regardless of whether it grew or shrank. The v3 pipeline correctly uses `change = carbon - prev_carbon` but its `prev_carbon` defaults to 0 forever (line 32), making classification meaningless.

6. **No CSV save in `app.py`:** Line 295 prints `"CSV saved: mrv_results.csv"` but the script never calls `df.to_csv()`. The CSV in the repo was generated by a previous run and is stale.

7. **`MinorProjectv3.0/pipeline/mrv_pipeline.py`:** Uses `np.random.choice([0, 1], ...)` (line 36) — generates fake satellite data. This is not connected to `app.py` at all. There are two parallel pipelines that don't talk to each other.

8. **`MinorProjectv3.0/requirements.txt`:** Empty file. Zero dependency management.

9. **`MinorProjectv3.0/README.md`:** Empty file. No documentation at all.

### GEE Implementation (gee.py)

```python
import ee

def initialize():
    ee.Initialize()

def get_mangrove_tiles(year: int) -> str:
    dataset = ee.ImageCollection("JAXA/ALOS/PALSAR/YEARLY/SAR") \
        .filterDate(f'{year}-01-01', f'{year}-12-31')
    
    # Use Global Mangrove Watch dataset
    gmw = ee.ImageCollection("projects/sat-io/open-datasets/GMW/annual") \
        .filter(ee.Filter.eq('year', year)) \
        .first()
    
    mangrove = gmw.select('b1').eq(1)
    
    vis_params = {
        'min': 0, 'max': 1,
        'palette': ['0d1117', '22c55e']  # dark bg, green mangrove
    }
    
    map_id = mangrove.getMapId(vis_params)
    return map_id['tile_fetcher'].url_format

def get_carbon_stats(year: int, region_bounds: list) -> dict:
    region = ee.Geometry.Rectangle(region_bounds)
    gmw = ee.ImageCollection("projects/sat-io/open-datasets/GMW/annual") \
        .filter(ee.Filter.eq('year', year)) \
        .first()
    
    pixel_area = ee.Image.pixelArea()
    mangrove_area = gmw.select('b1').eq(1).multiply(pixel_area)
    
    stats = mangrove_area.reduceRegion(
        reducer=ee.Reducer.sum(),
        geometry=region,
        scale=25,
        maxPixels=1e13
    )
    
    area_m2 = stats.getInfo()['b1']
    area_ha = area_m2 / 10000
    carbon_tons = area_ha * 392  # IPCC blue carbon default: 392 tCO₂/ha
    
    return {'area_ha': area_ha, 'carbon_tons': carbon_tons, 'year': year}
```

> **Note on the carbon constant:** Use **392 tCO₂/ha** (IPCC Tier 1 mangrove default) instead of the current arbitrary `100` or `1000`. This is critical for scientific credibility.

## 5. FILE UPLOAD + PROCESSING DESIGN

### Flow

```
Browser: User selects .tif file(s)
    │  multipart/form-data POST
    ▼
FastAPI: POST /api/upload
    │
    ├── Validate: extension == .tif, size < 500MB
    ├── Save to: uploads/{session_id}/{filename}
    ├── Return: { "job_id": "abc123", "status": "queued" }
    │
    ▼
FastAPI: POST /api/process/{job_id}    (or background task)
    │
    ├── pipeline.process_tif(path)     ← refactored from app.py
    ├── Save to DB: INSERT INTO results ...
    ├── If risk == "Severe Loss": alerts.send_telegram()
    └── Return: { "status": "complete", "result": {...} }
    │
    ▼
React: Poll GET /api/jobs/{job_id} every 2s
    │   OR listen on WebSocket /ws/job/{job_id}
    ▼
Dashboard updates with new result row
```

### FastAPI Upload Endpoint

```python
from fastapi import FastAPI, UploadFile, File, BackgroundTasks
import shutil, uuid, os

app = FastAPI()
UPLOAD_DIR = "uploads"

@app.post("/api/upload")
async def upload_tif(
    file: UploadFile = File(...),
    background_tasks: BackgroundTasks = None
):
    if not file.filename.endswith(".tif"):
        raise HTTPException(400, "Only .tif files accepted")
    
    job_id = str(uuid.uuid4())
    save_path = f"{UPLOAD_DIR}/{job_id}/{file.filename}"
    os.makedirs(os.path.dirname(save_path), exist_ok=True)
    
    with open(save_path, "wb") as f:
        shutil.copyfileobj(file.file, f)
    
    background_tasks.add_task(process_job, job_id, save_path)
    return {"job_id": job_id, "status": "processing"}

async def process_job(job_id: str, path: str):
    result = pipeline.process_tif(path)
    db.save_result(job_id, result)
    if result["risk"] == "Severe Loss":
        alerts.send_telegram(result)
```

## APPENDIX: FILENAME PARSING REFERENCE

The GMW dataset uses a standard naming convention. Use this to extract spatial + temporal context:

```python
import re

def parse_gmw_filename(filename: str) -> dict:
    """
    GMW_N22E089_2008_v3.tif
    → lat=22, lon=89, year=2008
    """
    match = re.match(r'GMW_([NS])(\d+)([EW])(\d+)_(\d{4})_v\d+\.tif', filename)
    if not match:
        return {}
    ns, lat, ew, lon, year = match.groups()
    return {
        'lat': int(lat) * (1 if ns == 'N' else -1),
        'lon': int(lon) * (1 if ew == 'E' else -1),
        'year': int(year)
    }
```

This enables:
- Grouping results by geographic region
- Time-series analysis per location
- Map marker placement
- Filtering by year in the UI
